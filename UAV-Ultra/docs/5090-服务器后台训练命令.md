# UAV-Ultra 5090 服务器后台训练命令

## 结论

重构后的 `UAV-Ultra` 不再使用旧入口：

```bash
python -m Main.train.train_*
python -m Main.evaluate.evaluate_*
```

统一改为：

```bash
python -m src.training.runner <algorithm>
python -m src.evaluation.runner <algorithm>
```

可以继续用 `nohup ... > runs/logs/launcher_*.out 2>&1 & echo $!` 在 5090 服务器后台跑。后台命令建议用 `python -u`，避免 stdout/stderr 被缓冲后日志长时间不刷新。训练 runner 会为每次运行先创建独立实验目录，并把 stdout/stderr 同步写入该目录下按现有规则命名的日志，例如 `runs/experiment-data/qmix_exp1/qmix_wm_block_jp_3000_seed0.out`；中途报错的完整 traceback 也会进入这个文件。如果没有先 `conda activate marl`，优先用下面的 `bash -lc 'conda activate ... && exec python -u ...'` 模板，不建议把长训练直接包在 `conda run` 里。

需要注意三点：

- 不要直接照抄旧版的 `--learn-every / --lr-q / --wm-batch-size` 等参数；这些现在主要放在 `src/config/defaults/*.yaml` 和 `src/config/defaults/presets/*.yaml` 里。
- `--steps` 一般不写；默认从 `src/config/defaults/env.yaml` 的 `max_episode_steps` 读取。临时短跑或 debug 才加 `--steps <N>`。
- 5090 + CUDA 训练建议用 `--start-method fork`。旧版常用 `fork`，但新版默认就是 `spawn`，对 CUDA 多进程更稳。

## 启动前准备

把 `PROJECT` 改成服务器上实际的 `UAV-Ultra` 目录。

```bash

# 如果服务器只单独放了 UAV-Ultra，用这一行替换上面那行：

cd "$PROJECT"
git pull --ff-only
python -m pip install -e .
mkdir -p runs/logs
```

如果没有在当前 shell 里先 `conda activate marl`，后台启动时用这个模板，避免 `conda run` 捕获输出：

默认训练产物写到：


## Preset 对照表

重构后用 preset 表达旧训练脚本的语义：

| 旧脚本 | 新命令 |
|---|---|
| `train_iql.py` | `python -m src.training.runner iql --preset iql_baseline` |
| `train_vdn.py` | `python -m src.training.runner vdn --preset vdn_baseline` |
| `train_qmix.py` | `python -m src.training.runner qmix --preset qmix_plain_baseline` |
| `train_qplex.py` | `python -m src.training.runner qplex --preset qplex_baseline` |
| `train_mappo.py` | `python -m src.training.runner mappo --preset mappo_baseline` |
| `train_qmix_value_expansion.py` | `python -m src.training.runner qmix --preset qmix_wm_concurrent_baseline` |
| _Ultra-only: concurrent + JP_ | `python -m src.training.runner qmix --preset qmix_wm_concurrent_jp_baseline` |
| `train_qmix_wm_alternating.py` | `python -m src.training.runner qmix --preset qmix_wm_block_baseline` |
| `train_qmix_wm_alternating_jammer_prediction.py` | `python -m src.training.runner qmix --preset qmix_wm_block_jp_baseline` |
| `train_qmix_wm_alternating_jammer_prediction_critic_stable.py` | `python -m src.training.runner qmix --preset qmix_wm_block_jp_cs_baseline` |

`UAV-Ultra` 没有独立 `train_world_model` 入口；world model 作为 QMIX callback 组件由 preset 自动挂载。

DQN 族与 QMIX+WM 系列 preset 与 baseline `train_*.py` 字段级对齐：

```text
n_episode=3000, num_envs=16, batch_size=512
learn_every=4, updates_per_learn=1
lr_actor=1e-3, lr_q=1e-3, lr_mixer=1e-3
max_grad_norm=10.0 (MAPPO 0.5), use_amp=true, start_method=fork
```

`lr_*`、`max_grad_norm`、`learn_every/updates_per_learn` 取自 baseline argparse default / 已归档 5090 `run_config.json`；`num_envs/batch_size/n_episode/start_method` 保留 5090 实测吞吐设置（baseline 5090 命令同值）。

`--amp / --no-amp` 可以临时覆盖 preset 中的 `use_amp`。默认不写时沿用 preset；3090/4090/5090 上建议先用 preset AMP 路径，若遇到数值不稳或想做基准对照，加 `--no-amp`。

```bash
python -m src.training.runner qmix \
  --preset qmix_plain_baseline \
  --episodes 1 \
  --steps 2 \
  --num-envs 1 \
  --batch-size 2 \
  --device cpu \
  --start-method fork \
  --no-amp \
  --no-save
```

中断续训使用 `--resume`。Stage 10 只恢复 trainer weights + callback state；episode counter、replay buffer、历史曲线不恢复，新 run 会生成独立 artifact，并在 `training_data.json.config.resume_from` 写入 checkpoint 绝对路径和 sha256。

```bash
nohup python -u -m src.training.runner qmix \
  --preset qmix_wm_block_jp_cs_baseline \
  --resume /path/to/qmix_weights.pth \
  --episodes 3000 \
  --device cuda \
  --num-envs 16 \
  --batch-size 512 \
  --start-method fork \
  --seed 0 \
  > "runs/logs/launcher_resume_qmix_wm_block_jp_cs_seed0_$(date +%Y%m%d_%H%M%S)_${RANDOM}.out" 2>&1 & echo $!
```

## QMIX + World Model concurrent 训练（Q + WM 同时更新）

对应旧版 `train_qmix_value_expansion.py`，新版用 `qmix_wm_concurrent_baseline` preset。注意：VE-style TD target（`α·G_λ + (1-α)·bootstrap`）在 `concurrent` 与 `block` 路径上共享；这条 preset 与下面的 `wm_block` preset 的区别只在 **WM 更新调度**——`concurrent` 每个 Q learn step 顺带跑 `wm_updates_per_learn` 次 WM；`block` 是 episode 级 50/50 相位切换。

```bash
from src.evaluation import run_evaluation

preset = load_experiment_preset("qmix_wm_block_jp_cs_baseline")
result = run_evaluation(
    "qmix",
    checkpoint_path=Path("/path/to/qmix_weights.pth"),
    algo_overrides=preset.algo,
    callback_overrides=preset.algo["callbacks"],
    episodes=100,
    num_envs=16,
    seed=0,
    device="cuda",
    start_method="spawn",
)
print("Output:", result.output_dir)
PY
echo $!
```

```bash
  --start-method fork \
  --seed 0 \
  > "runs/logs/launcher_eval_latest_qmix_seed0_$(date +%Y%m%d_%H%M%S)_${RANDOM}.out" 2>&1 & echo $!
```

## 多 seed 建议

正式论文主表建议至少跑 `5` 个 seed。每个 seed 改 `--seed` 和日志名即可：

```bash
nohup python -u -m src.training.runner qmix \
  --preset qmix_plain_baseline \
  --episodes 3000 \
  --device cuda \
  --num-envs 16 \
  --batch-size 512 \
  --start-method fork \
  --seed 1 \
  > "runs/logs/launcher_qmix_3000_seed1_$(date +%Y%m%d_%H%M%S)_${RANDOM}.out" 2>&1 & echo $!
```

一张 5090 上不要同时跑太多训练任务。建议一次只跑 `1` 个训练任务；heuristic/evaluation 相对轻，可以在训练空档批量跑。
